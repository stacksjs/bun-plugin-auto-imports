# Getting Started

Learn how to set up auto-imports in your Bun project.

## Installation

```bash
bun install -d bun-plugin-auto-imports
```

## Basic Usage

Register the plugin in your main entry file:

```typescript
// index.ts
import type { AutoImportsOptions } from 'bun-plugin-auto-imports'
import { plugin } from 'bun'
import { autoImports } from 'bun-plugin-auto-imports'

const options: AutoImportsOptions = {
  presets: ['solid-js'],
  imports: [{ name: 'z', from: 'zod' }],
  dts: './auto-imports.d.ts'
}

plugin(autoImports(options))

// Start your server
Bun.serve({
  fetch: handler,
  port: 3000
})
```

## Using Auto-Imported Modules

After configuration, you can use imports without explicit import statements:

```typescript
// server.ts
// 'z' is automatically imported from zod
const Body = z.object({
  msg: z.string(),
  count: z.number().optional()
})

export async function handler(req: Request) {
  try {
    const body = await req.json()
    const data = Body.parse(body)

    return new Response(`Received: ${data.msg}`)
  } catch (e) {
    return new Response('Invalid body', { status: 400 })
  }
}
```

## Configuration Options

### Presets

Use pre-configured imports for popular libraries:

```typescript
const options: AutoImportsOptions = {
  presets: ['solid-js', 'vue', 'react']
}
```

### Custom Imports

Import specific functions from any package:

```typescript
const options: AutoImportsOptions = {
  imports: [
    { name: 'z', from: 'zod' },
    { name: 'clsx', from: 'clsx' },
    { name: 'default', as: 'axios', from: 'axios' }
  ]
}
```

### Directory Scanning

Automatically import exports from directories:

```typescript
const options: AutoImportsOptions = {
  dirs: ['./src/utils', './src/composables']
}
```

### TypeScript Declarations

Generate type declarations for IntelliSense:

```typescript
const options: AutoImportsOptions = {
  dts: './src/auto-imports.d.ts'
}
```

## Generated Files

### auto-imports.d.ts

The plugin generates a TypeScript declaration file:

```typescript
// Generated by bun-plugin-auto-imports
export {}
declare global {
  const z: typeof import('zod')['z']
  const clsx: typeof import('clsx')['clsx']
}
```

### .eslint-auto-import.json

With ESLint integration enabled:

```json
{
  "globals": {
    "z": "readonly",
    "clsx": "readonly"
  }
}
```

## TypeScript Configuration

Add the generated declaration file to your `tsconfig.json`:

```json
{
  "compilerOptions": {
    "types": ["./auto-imports.d.ts"]
  },
  "include": ["./auto-imports.d.ts"]
}
```

Or reference it in a file:

```typescript
/// <reference path="./auto-imports.d.ts" />
```

## Example Project Structure

```
my-project/
├── src/
│   ├── index.ts      # Plugin registration
│   ├── server.ts     # Uses auto-imports
│   └── utils/        # Scanned for exports
├── auto-imports.d.ts # Generated declarations
└── tsconfig.json
```

## How It Works

1. **Plugin Registration**: The plugin registers with Bun's plugin system
2. **Scanning**: On setup, it scans configured directories for exports
3. **Injection**: When files are loaded, imports are automatically injected
4. **Type Generation**: Declaration files are generated for TypeScript
